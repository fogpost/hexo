## 正向代理和方向代理
代理就是中介
一个完整的请求是由: client(客户端) -> proxy(代理) -> server(服务端) 组成
正向代理: 顺着请求的方向进行的代理，即[代理服务器](https://zhida.zhihu.com/search?content_id=124566892&content_type=Article&match_order=1&q=%E4%BB%A3%E7%90%86%E6%9C%8D%E5%8A%A1%E5%99%A8&zhida_source=entity)它是由你配置为你服务，去请求目标服务器地址
正向代理的作用:
1. 访问原来无法访问的资源，如google
2. 可以做[缓存](https://zhida.zhihu.com/search?content_id=124566892&content_type=Article&match_order=1&q=%E7%BC%93%E5%AD%98&zhida_source=entity)，加速访问资源
3. 对客户端访问授权，上网进行认证
4. 代理可以记录用户访问记录（[上网行为管理](https://zhida.zhihu.com/search?content_id=124566892&content_type=Article&match_order=1&q=%E4%B8%8A%E7%BD%91%E8%A1%8C%E4%B8%BA%E7%AE%A1%E7%90%86&zhida_source=entity)），对外隐藏用户信息

反向代理: 跟正向代理相反,它是为目标服务器进行服务的,但是请求的流程还是: clieng -> proxy -> server 代理远端服务器提供响应
反向代理的作用:  
1. 保证内网的安全，阻止web攻击，大型网站，通常将反向代理作为公网访问地址，Web服务器是内网。
2. [负载均衡](https://zhida.zhihu.com/search?content_id=124566892&content_type=Article&match_order=1&q=%E8%B4%9F%E8%BD%BD%E5%9D%87%E8%A1%A1&zhida_source=entity)，通过反向代理服务器来优化网站的负载

正向代理即是客户端代理, 代理客户端, 服务端不知道实际发起请求的客户端.
反向代理即是服务端代理, 代理服务端, 客户端不知道实际提供服务的服务端.
