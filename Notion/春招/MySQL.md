truncate / delete / drop
delete 是逐行删除数据，可以加 where 条件，并且支持事务回滚；  
truncate 是清空整张表，速度更快，会重置自增 ID，但不能回滚；  
drop 是直接删除表结构和数据，相当于把表删除。

| 操作     | delete | truncate | drop |
| ------ | ------ | -------- | ---- |
| 删除对象   | 删除数据   | 清空数据     | 删除表  |
| 是否可回滚  | 可以     | 不可以      | 不可以  |
| 是否记录日志 | 逐行记录   | 很少       | 很少   |
| 是否重置自增 | 不会     | 会        | 表没了  |
| 速度     | 慢      | 快        | 最快   |

InnoDB 支持事务、行级锁和外键，并且支持崩溃恢复，是现在 MySQL 默认的存储引擎。  
MyISAM 不支持事务，只支持表锁，读性能较好，但可靠性较低，所以现在大多数项目都会使用 InnoDB。

|特性|InnoDB|MyISAM|
|---|---|---|
|事务|支持|不支持|
|行锁|支持|表锁|
|外键|支持|不支持|
|崩溃恢复|支持|不支持|
|性能|写更好|读更快|

索引类型
1️⃣ **主键索引（Primary Key）**  
2️⃣ **唯一索引（Unique）**  
3️⃣ **普通索引（Index）**  
4️⃣ **联合索引（Composite Index）**  
5️⃣ **全文索引（Fulltext）**

MySQL 常见索引有主键索引、唯一索引、普通索引和联合索引等。  
在项目中通常会使用主键索引和联合索引，比如在查询频繁的字段上建立索引，提高查询效率

慢查询日志
使用过 MySQL 慢查询日志来排查 SQL 性能问题，通过设置 slow_query_log 和 long_query_time 来记录执行时间较长的 SQL，然后根据日志对 SQL 进行优化，比如增加索引或优化查询语句。
MySQL 支持 **慢查询日志**
用于定位 **SQL性能问题**
