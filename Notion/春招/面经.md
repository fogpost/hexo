Sql注入攻击原理及防范  
**SQL注入** 是攻击者通过输入恶意 SQL 语句，改变数据库查询逻辑，从而获取、修改或删除数据库数据的攻击方式，程序把用户输入直接拼接进 SQL 语句。
防御：参数化查询、输入验证、最小权限、使用ROM框架

Sql注入有回显和无回显  
有回显 union\报错
无回显 bool和时间

Xss攻击原理，分类，及防范  
攻击者向网页注入恶意脚本（通常是 JavaScript），在其他用户浏览页面时执行
- 窃取 Cookie
- 劫持 Session
- 钓鱼页面
- 执行恶意操作

分类：
存储型XSS：攻击代码存储在服务器数据库
```js
<script>alert(document.cookie)</script>
```


反射型XSS：攻击代码在url参数中（钓鱼链接）
```
http://site.com/search?q=<script>alert(1)</script>
```


DOM型XSS：发生在前端js操作DOM时
```js
document.write(location.hash)
```

XSS防御：
输入过滤:
```
<script>
onerror
javascript:
```
输出编码:
```js
<script> → &lt;script&gt;
```
CSP策略:
http头：Content-Security-Policy
HttpOnly Cookie：
防止js读取cookie

文件上传漏洞原理  
网站允许用户上传文件，但 **没有限制文件类型**。

常见绕过：
文件后缀 shell.php.jpg
MIME绕过 Content-Type: image/jpeg
双写绕过 shell.pphphp
.htaccess绕过

防御：
白名单限制 jpg、png、gif
MIME检测 检测真实文件类型
重命名文件
上传目录禁止执行：
Nginx:
```php
location /upload {
    deny all;
}
```


钓鱼邮件原理  
攻击者伪装成 **可信机构**，诱骗用户提供敏感信息。
攻击方式：
假登录：paypal-security-login.com
恶意附件：invoice.doc宏病毒
链接跳转
防御：不点击邮件链接、检查域名、邮件网关过滤、SPF/DKIM/DMARC

TCP和UDP区别，应用场景 

|特点|TCP|UDP|
|---|---|---|
|连接|面向连接|无连接|
|可靠性|可靠|不可靠|
|顺序|有序|无序|
|速度|慢|快|
|重传|有|无|
TCP：HTTP、HTTPS、FTP、SSH 可靠
UDP：DNS、视频流、Voip、游戏 低延迟

RIP，OSPF，BGP4路由协议  
RIP：距离矢量协议：最大跳数15、收敛慢 小型网络
OSPF：链路状态协议：收敛快 大型网络
BGP4：边界网关协议：互联网as路由 ISP之间路由
