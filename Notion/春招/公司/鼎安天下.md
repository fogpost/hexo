# 安全驻场与 AI 智能体岗位面试冲刺指南

**Subject:** 网络安全实战与 AI 赋能安全 (AI+Security)

**Topics:**

- AI 智能体 (Security Agent) 与 RAG 落地
    
- SOC 自动化研判与应急响应 (IR)
    
- 红蓝对抗 (HW) 与内网渗透核心目标
    
- 安全开发中的代码安全与 Vibe Coding 风险
    

**Summary:** 本指南旨在帮助面试者应对融合了传统网络安全服务（安服）与最新 AI 编程/智能体工具链的复合型岗位。重点在于如何利用 AI 工具（如 Dify, LangChain）优化传统的安全监控、渗透测试及漏洞修补流程，同时掌握在大型企业（如顺丰）驻场环境下的标准作业程序 (SOP)。

**Key Concepts:**

- **RAG (检索增强生成):** 通过将私有安全知识库（如历史工单、厂商文档）转化为向量，解决大模型在安全专业领域下的“幻觉”问题，确保输出的准确性。
    
- **Workflow 自动化编排:** 利用 Dify 等平台的低代码流处理能力，将“告警接入-情报查询-AI 研判-自动封禁”串联成自动化闭环，降低 SOC 一线压力。
    
- **隔离优先原则:** 在应急响应（尤其是勒索病毒场景）中，物理或逻辑隔离受感染资产是止损的最高优先级。
    
- **安全域控 (DC) 攻防:** 内网渗透的终极目标通常是控制域控，以实现对企业全局资产的权限掌控。
    
- **参数化查询:** 抵御 SQL 注入的最根本技术手段，通过将数据与执行逻辑分离实现防护。
    

**Vocabulary List:**

- **Vibe Coding:** 指开发者主要通过与 AI（如 Cursor, Claude）对话、利用自然语言描述需求来生成和调试代码的开发模式。
    
- **SOC (Security Operations Center):** 安全运营中心，负责对企业网络安全态势进行实时监控、研判和处置。
    
- **HW (Hiding Wolf/护网):** 指代高强度的红蓝实战攻防演练，通常分为攻击方（红队）和防守方（蓝队）。
    
- **Dify/LangChain:** 当前主流的 AI 应用开发框架/平台，用于构建智能体和自动化工作流。
    
- **WAF (Web Application Firewall):** Web 应用防火墙，用于过滤和阻断针对 Web 业务的恶意攻击流量。
    

**Key Questions:**

- 如何在确保业务连续性的前提下，利用 AI 实现 SOC 告警的“去误报”？
    
- 当 AI 生成的安全脚本存在逻辑漏洞时，作为安全工程师应如何建立审计流程？
    
- 在驻场环境中，如果发现 AI 助手泄露了公司内部 API Key，你应该采取哪些应急措施？
    
- 描述一次你利用 Python 或 AI 工具显著提升渗透测试效率的经历。
