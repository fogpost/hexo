
## 删除痕迹
**合法授权测试 + 安全审计**

在正规的渗透测试中，一般不会恶意删除系统日志，因为日志是企业审计和溯源的重要证据。渗透测试通常是在授权范围内进行，测试完成后主要是**清理测试产生的工具、账户、后门以及测试数据**，并将相关情况记录在报告中。


## 清理工作
### 清理测试工具
- Webshell
- 扫描工具
- 临时脚本
- 上传的提权工具
### 删除测试账号
userdel testuser
net user testuser /del
### 删除后门或者计划任务
- 计划任务
- 反弹 shell
- 持久化脚本
### 清理测试产生的文件
- 上传的木马文件
- 测试数据
- 临时配置
### 保留必要的日志用于报告
- 利用过程
- 时间线
- 攻击路径
- 证据截图

从攻击者角度来说，可能会尝试清理日志，比如删除 Linux 的 `/var/log` 日志或者清空 Windows 事件日志，但这种行为通常会留下异常，因此企业通常会通过集中日志系统或 SIEM 平台进行日志备份和监控。
## 攻击者的日志清除
Linux：
- `/var/log/auth.log`
- `/var/log/secure`
- `/var/log/syslog`
Windows：
- Security Log
- System Log
- Application Log
攻击者可能会尝试：
- 删除日志文件
- 修改日志
- 清空日志
通常会留下异常痕迹
- 日志突然中断
- 时间戳异常
- 审计记录缺失
