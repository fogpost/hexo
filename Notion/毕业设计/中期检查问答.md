## 1. 项目概况

- 问：这个项目做什么？
- 答：基于 PCAP 报文分析的“网络流量异常检测与可视化”系统。实现流量抓取、协议解析、统计、检测引擎、前端展示等一体化流程。
    
- 问：目标用户是谁？
- 答：网络运维、安全团队、教学实验及个人学习，目标是快速检测 DDoS / ARP 欺骗 /异常连接，提供桌面 + Web 可视化界面。

## 3. 关键技术与架构设计
- 数据流：
    - 抓包 -> 解析（协议、5元组） -> 统计 -> 检测 -> 可视化/告警。
- 异常检测策略（推测）：
    - SYN flood 速率阈值。
    - ARP 假冒同一 MAC。
    - ICMP Flood 包量超限。
- 前端/桌面：
    - Vue + Wails 将 Go 后端绑定到前端。
    - 内嵌静态资源（`//go:embed all:frontend/dist`）。

## 5. 风险与下一步（一定要有）
- 风险：
    1. 大流量下解析性能未验证（内存/CPU）。
    2. 特征规则硬编码，易漏非典型攻击。
    3. 跨平台桌面打包（Windows/Mac/Linux）兼容需补。
- 解决方案：
    - 引入流式、批处理/滑动窗口。
    - 可配置 JSON 规则 + 机器学习模型（后期）。
    - 使用 CI 打包测试，计划调用 wails build。
