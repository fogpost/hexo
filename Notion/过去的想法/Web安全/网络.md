# NAT

NAT 的基本工作原理是，将局域网内的私有 IP 地址（通常在 192.168.x.x、10.x.x.x 等范围）转换为一个公共的 IP 地址，从而使多个设备可以共享一个公共 IP 地址访问互联网。NAT 还会负责在返回的流量中反向映射，确保数据返回到正确的设备

在 WSL 中使用 NAT 网络模式时，的确可能会遇到主机无法直接通过 WSL 的 IP 地址访问 Docker 容器或 WSL 内的服务的问题。这是因为 Docker 和 WSL 使用的 NAT 网络模式通常使得容器的网络和宿主机（Windows）的网络隔离开来

**NAT 网络隔离**：

- 在 WSL 2 中，Docker 通常使用 NAT 网络模式，这意味着容器和宿主机之间通过一个虚拟的网络接口连接，而容器的 IP 地址是一个虚拟的私有 IP 地址，外部网络（包括宿主机）无法直接访问这些 IP 地址。
- 宿主机通过 Docker 守护进程与容器通信，但容器的内部 IP 地址并不直接暴露给宿主机

**WSL 和 Docker 网络层的不同**：

- WSL 和 Docker 都运行在 Windows 的 Hyper-V 虚拟化技术上，二者的网络是通过虚拟化的桥接或 NAT 网络进行连接的，因此 WSL 和 Docker 容器的 IP 地址通常是私有的，并且不能直接通过宿主机的 IP 地址访问
